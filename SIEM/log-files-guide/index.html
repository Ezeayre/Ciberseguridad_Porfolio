📊 Log Files Guide

Guía interactiva para leer, analizar y auditar logs en Linux y Windows

📖 ¿Por qué son importantes los Logs?

Los logs son el diario del sistema. Registran todo lo que sucede: desde un inicio de sesión exitoso hasta un error crítico que podría comprometer la seguridad.

En esta guía aprenderás:

  • 🔍 Dónde encontrar los logs en Linux y Windows
  • 📖 Cómo leerlos e interpretar su estructura
  • 🛠️ Qué herramientas usar para analizarlos eficientemente
  • 🎯 Casos prácticos de seguridad y troubleshooting

Ya seas administrador de sistemas, analista de seguridad o desarrollador, entender los logs es una habilidad fundamental para mantener sistemas saludables y seguros.

📖 Cómo Leer un Log

Leer un log es como leer un diario: necesitas entender el formato, identificar lo importante y saber qué buscar. Esta es la habilidad fundamental que dominarás con esta guía.

🔍 Estructura de un Log

Un log típico tiene 4 componentes principales:

2026-06-11 14:35:22 [INFO] Usuario juan inició sesión desde IP 192.168.1.100 2026-06-11 14:36:45 [WARNING] Acceso denegado a /etc/passwd por usuario guest 2026-06-11 14:37:10 [ERROR] Conexión a base de datos fallida

1. Timestamp

▼

2026-06-11 14:35:22

Cuándo ocurrió el evento. Es crucial para:

  • Correlacionar eventos en el tiempo
  • Identificar patrones temporales
  • Investigar incidentes específicos

2. Nivel de Severidad

▼

[INFO] [WARNING] [ERROR]

Qué tan importante es el evento:

  • DEBUG: Información de depuración (solo desarrollo)
  • INFO: Información general (operación normal)
  • WARNING: Advertencia (no crítico pero requiere atención)
  • ERROR: Error (necesita investigación)
  • CRITICAL/FATAL: Error crítico (sistema en riesgo)

3. Fuente/Componente

▼

Usuario juan Conexión a base de datos

Quién o qué generó el evento:

  • Usuario específico
  • Servicio o aplicación
  • Componente del sistema

4. Mensaje

▼

inició sesión desde IP 192.168.1.100

Qué pasó exactamente:

  • Acción realizada
  • Resultado de la acción
  • Contexto adicional (IP, archivo, etc.)

💡 Tips para Leer Logs Eficientemente

🎯 Empieza por ERROR y CRITICAL

▼

Filtra primero por errores críticos. Estos son los que necesitan atención inmediata.

⏰ Usa el timestamp inteligentemente

▼

Cuando investigues un incidente, filtra por el rango de tiempo del problema.

🔗 Correlaciona múltiples logs

▼

Un error en una aplicación puede tener su causa en un log de sistema.

📊 Busca patrones

▼

Un error aislado puede ser normal. 100 errores del mismo tipo en 5 minutos es un problema.

📝 Ejemplos de Análisis de Logs

Ejemplo 1: Intento de intrusión

▼
2026-06-11 14:35:22 [ERROR] Failed password for invalid user admin from 192.168.1.100 2026-06-11 14:35:25 [ERROR] Failed password for invalid user admin from 192.168.1.100 2026-06-11 14:35:28 [ERROR] Failed password for invalid user admin from 192.168.1.100

Análisis: 3 intentos fallidos de login como "admin" desde la misma IP en 6 segundos. Posible ataque de fuerza bruta.

Ejemplo 2: Error de aplicación

▼
2026-06-11 14:40:15 [ERROR] Database connection failed: Connection timeout 2026-06-11 14:40:16 [WARNING] Retrying connection to database... 2026-06-11 14:40:18 [INFO] Database connection established

Análisis: Error de conexión a base de datos, pero el sistema se recuperó automáticamente. Puede ser un problema de red temporal.

Ejemplo 3: Acceso no autorizado

▼
2026-06-12 02:15:44 [ALERT] User 'root' logged in from 203.0.113.50 (SSH) 2026-06-12 02:16:12 [WARNING] Privilege escalation attempt detected 2026-06-12 02:16:30 [ERROR] Access denied to /etc/shadow

Análisis: Root login desde IP desconocida a las 2 AM + intento de escalación. Comportamiento ANORMAL. Investigar inmediatamente.

Ejemplo 4: Servicio caído y recuperación

▼
2026-06-12 10:45:32 [CRITICAL] Apache service stopped unexpectedly 2026-06-12 10:45:33 [ERROR] Segmentation fault in module mod_ssl 2026-06-12 10:45:35 [INFO] Apache service restarting... 2026-06-12 10:45:38 [INFO] Apache service started successfully

Análisis: Crash por segmentation fault en SSL. El problema se recuperó automáticamente. Revisar changelog del módulo mod_ssl.

Ejemplo 5: Actividad sospechosa de usuario

▼
2026-06-12 15:20:10 [INFO] User 'john' executed: sudo cat /etc/shadow 2026-06-12 15:20:15 [INFO] User 'john' executed: sudo cp /etc/shadow /tmp/backup.txt 2026-06-12 15:20:45 [INFO] User 'john' attempted SSH to 10.0.0.200 (external host)

Análisis: Usuario 'john' leyendo archivos sensibles y copiándolos. Intento de conexión a host externo. Posible compromiso. ALERTAR seguridad.

Ejemplo 6: Evento Windows - Múltiples fallos de logon

▼
EventID: 4625 | Time: 2026-06-12 09:15:32 | User: admin | Source IP: 192.168.1.50 EventID: 4625 | Time: 2026-06-12 09:15:45 | User: admin | Source IP: 192.168.1.50 EventID: 4625 | Time: 2026-06-12 09:15:58 | User: admin | Source IP: 192.168.1.50 EventID: 4624 | Time: 2026-06-12 09:16:05 | User: admin | Source IP: 192.168.1.50 | Logon Type: 3

Análisis: 3 fallos de logon (4625) seguidos de 1 éxito (4624) desde la misma IP. Probablemente contraseña descifrada tras fuerza bruta. REVISAR INMEDIATAMENTE.

🌐 General (Aplica a ambos sistemas)

¿Qué es un Log? ▼

Registro de eventos que ocurren en un sistema, como un diario del computador

Analogía:

Si tu computadora fuera una persona, los logs serían su diario personal donde anota todo lo que hizo cada día.

Características de un log:
  • Timestamp: Cuándo pasó (fecha y hora)
  • Nivel de severidad: Qué tan importante es (Error, Warning, Info)
  • Mensaje: Qué pasó exactamente
  • Fuente: De dónde vino el evento
Logs de Sistema ▼

Registran eventos del sistema operativo: arranques, apagados, errores de hardware

En Linux:
/var/log/syslog (Debian/Ubuntu)
/var/log/messages (RedHat/CentOS)
En Windows:
Visor de Eventos → System
Logs de Seguridad ▼

Registran eventos de seguridad: intentos de logon, cambios de permisos

En Linux:
/var/log/auth.log
/var/log/secure
En Windows:
Visor de Eventos → Security
Logs de Aplicaciones ▼

Registran eventos de aplicaciones específicas: errores, advertencias

En Linux:
/var/log/apache2/ (Apache)
/var/log/mysql/ (MySQL)
En Windows:
Visor de Eventos → Application

cat - Ver contenido ▼

Muestra todo el contenido de un archivo de log

Sintaxis:
cat [archivo]
Ejemplos:
cat /var/log/auth.log # Muestra todo el contenido
cat /var/log/syslog | grep 'error' # Busca errores
Cuándo usarlo:

Para ver rápidamente qué hay dentro de un log

tail - Ver últimas líneas ▼

Muestra las últimas líneas de un log, ideal para ver eventos recientes

Sintaxis:
tail [opciones] [archivo]
Ejemplos:
tail -10 /var/log/auth.log # Últimas 10 líneas
tail -f /var/log/auth.log # En tiempo real (live)
Cuándo usarlo:

Para ver los eventos más recientes o monitorear en vivo

head - Ver primeras líneas ▼

Muestra las primeras líneas de un log

Sintaxis:
head [opciones] [archivo]
Ejemplos:
head -20 /var/log/syslog # Primeras ️20 líneas
Cuándo usarlo:

Para ver el inicio de un log

grep - Buscar patrones ▼

Busca patrones de texto en archivos de logs

Sintaxis:
grep [opciones] 'patrón' archivo
Ejemplos:
grep 'Failed password' /var/log/auth.log # Busca intentos fallidos
grep -i 'error' /var/log/syslog # Ignora mayúsculas
grep -n 'error' /var/log/syslog # Muestra número de línea
grep -c 'error' /var/log/syslog # Cuenta ocurrencias
Cuándo usarlo:

Constantemente para buscar en logs y archivos

less - Ver página por página ▼

Navega por un log página por página

Sintaxis:
less [archivo]
Ejemplos:
less /var/log/auth.log # Abre navegador
# Space = siguiente página
# q = salir
# /texto = buscar
Cuándo usarlo:

Para navegar logs largos sin saturar la terminal

awk - Procesar texto ▼

Extrae columnas específicas y procesa texto avanzado

Sintaxis:
awk '{print $1, $2, $3}' archivo
Ejemplos:
awk '{print $1, $2, $3}' /var/log/auth.log # Extrae primeras 3 columnas
awk '/error/ {print}' /var/log/syslog # Muestra líneas con error
Cuándo usarlo:

Para extraer información específica de logs

sed - Reemplazar texto ▼

Reemplaza o modifica texto en logs

Sintaxis:
sed 's/patron/reemplazo/g' archivo
Ejemplos:
sed 's/error/ERROR/g' /var/log/syslog # Reemplaza error por ERROR
Cuándo usarlo:

Para modificar o formatear salida de logs

sort & uniq - Agrupar eventos ▼

Agrupa y cuenta eventos duplicados en logs

Sintaxis:
grep 'patron' archivo | sort | uniq -c
Ejemplos:
grep 'Failed' /var/log/auth.log | sort | uniq -c # Cuenta intentos fallidos por IP
awk '{print $1}' /var/log/auth.log | sort | uniq -c # Cuenta por fecha
Cuándo usarlo:

Para analizar patrones y frecuencias en logs

wc - Contar líneas ▼

Cuenta líneas, palabras y caracteres en logs

Sintaxis:
wc [opciones] archivo
Ejemplos:
grep 'Failed' /var/log/auth.log | wc -l # Cuenta intentos fallidos
wc -l /var/log/syslog # Total de líneas
Cuándo usarlo:

Para obtener estadísticas de logs

Pipes avanzados ▼

Combina múltiples comandos para análisis complejos

Sintaxis:
comando1 | comando2 | comando3
Ejemplos:
grep '2026-06-11' /var/log/syslog | grep 'ERROR' | wc -l # Errores en fecha específica
cat /var/log/auth.log | grep 'sudo' | grep 'COMMAND' # Comandos sudo ejecutados
Cuándo usarlo:

Para operaciones complejas encadenando comandos

Ejemplos Prácticos - Linux ▼

Casos de uso reales para análisis de logs en Linux

Casos de uso:
  • Investigar intrusiones: Buscar IPs sospechosas en auth.log
  • Auditoría: Ver quién ejecutó comandos sudo
  • Troubleshooting: Encontrar errores en syslog por fecha
  • Seguridad: Monitorear accesos a archivos sensibles
Ejemplos:
grep 'Failed password' /var/log/auth.log | tail -20 # Intentos fallidos recientes
grep 'sudo' /var/log/auth.log | grep 'COMMAND' # Quién usó sudo
grep 'passwd' /var/log/auth.log # Cambios en archivos sensibles
grep '2026-06-11' /var/log/syslog | grep 'ERROR' | wc -l # Errores en fecha específica

PowerShell - Get-EventLog ▼

Comando PowerShell para leer logs de eventos (clásico)

Sintaxis:
Get-EventLog -LogName [nombre] -Newest [n]
Ejemplos:
Get-EventLog -LogName Security -Newest 10 # Últimos 10 eventos
Get-EventLog -LogName Security -InstanceId 4625 # Logons fallidos
Get-EventLog -LogName Security | Export-Csv logs.csv # Exportar a CSV
Cuándo usarlo:

Para análisis básico de logs en Windows

PowerShell - Get-WinEvent ▼

Comando PowerShell moderno para leer logs de eventos (más potente)

Sintaxis:
Get-WinEvent -LogName [nombre] -FilterHashtable @{...}
Ejemplos:
Get-WinEvent -LogName Security -MaxEvents 10 # Últimos 10 eventos
Get-WinEvent -FilterHashtable @{LogName='Security'; Id=4625} # Logons fallidos
Get-WinEvent -LogName System | Where-Object {$_.Level -eq 2} # Solo errores
Cuándo usarlo:

Para análisis avanzado de logs en Windows (más eficiente que Get-EventLog)

PowerShell - Select-String ▼

Busca patrones de texto en archivos (similar a grep)

Sintaxis:
Select-String -Path [archivo] -Pattern [patrón]
Ejemplos:
Select-String -Path C:\logs\app.log -Pattern 'error' # Busca errores
Select-String -Path C:\logs\*.log -Pattern 'failed' -Context 2 # Con contexto
Get-Content C:\logs\app.log | Select-String 'warning' # Con pipe
Cuándo usarlo:

Para buscar patrones en archivos de log de texto

Wevtutil ▼

Herramienta nativa de línea de comandos para logs de eventos

Sintaxis:
wevtutil [comando] [opciones]
Ejemplos:
wevtutil el # Lista todos los logs
wevtutil qe Security /c:10 /rd:true # Últimos 10 eventos de Security
wevtutil qe Security /q:"*[System[(EventID=4625)]]" # Filtra por Event ID
Cuándo usarlo:

Para análisis rápido sin PowerShell

PowerShell - Exportar Logs ▼

Exportar logs a diferentes formatos para análisis

Sintaxis:
Get-EventLog | Export-Csv / Export-Clixml
Ejemplos:
Get-EventLog -LogName Security | Export-Csv security.csv -NoTypeInformation
Get-EventLog -LogName Application | Export-Clixml app.xml
Get-WinEvent -LogName System | ConvertTo-Json | Out-File system.json
Cuándo usarlo:

Para exportar logs para análisis externo o auditoría

PowerShell - Análisis Avanzado ▼

Técnicas avanzadas de análisis de logs con PowerShell

Ejemplos:
# Agrupar eventos por Event ID
Get-EventLog -LogName Security | Group-Object EventId -NoElement | Sort-Object Count -Descending
# Filtrar por rango de tiempo
$start = Get-Date '2026-06-11 08:00'
$end = Get-Date '2026-06-11 18:00'
Get-EventLog -LogName Security | Where-Object {$_.TimeGenerated -gt $start -and $_.TimeGenerated -lt $end}
# Buscar IPs sospechosas en logs de seguridad
Get-EventLog -LogName Security -InstanceId 4625 | ForEach-Object {$_.Message -match 'Source Network Address: (\d+\.\d+\.\d+\.\d+)'; $Matches[1]} | Group-Object | Sort-Object Count -Descending
Cuándo usarlo:

Para análisis complejos y automatización

💡 Tips Importantes y Buenas Prácticas ▼

🔍 Al Analizar Logs:
  • Busca patrones anormales (múltiples errores, IPs raras)
  • Siempre correlaciona eventos con timestamps
  • Documenta TODO lo que investigues
  • Guarda resultados en archivo: comando > analisis.txt
🛡️ Seguridad en Logs:
  • Los logs contienen datos sensibles (contraseñas, IPs, usuarios)
  • Protege archivos de logs con permisos restringidos
  • Mantén logs por tiempo prolongado (auditoría)
  • Centraliza logs en servidor seguro (SIEM)
⚡ Eficiencia:
  • Usa herramientas (grep, awk, sed) en línea
  • Crea alias para comandos frecuentes
  • Automatiza búsquedas con scripts
  • Usa tail -f para monitoreo en tiempo real
🚨 Eventos Críticos a Monitorear:
  • Multiple failed logins (fuerza bruta)
  • Usuarios creados/eliminados inesperadamente
  • Cambios en permisos de archivos sensibles
  • Acceso a /etc/shadow o archivos del sistema
